# Import Service — монорепозиторий

**Назначение.** Программный комплекс для сервиса импорта: клиенты работают с заявками и связанными процессами через мобильное приложение; администрирование — через веб-админку; данные и интеграции обслуживает серверный API.

## Возможности (кратко)

- Учёт и работа с автомобилями и заявками в контексте импорта (списки, статусы, детали заявок).
- Таможенные заявки и сопутствующие сценарии (черновики, файлы, переписка по заявке).
- Авторизация и профиль организации и пользователя в приложении.
- Интеграция бэкенда с учётной системой **1С** (синхронизация пользователей и организаций и интеграционные точки согласно настройкам окружения).
- Чат по заявке с обменом сообщениями в реальном времени (WebSocket).

## Состав репозитория

| Каталог | Роль |
|--------|------|
| `import_service_app` | Мобильное приложение для клиентов (**Flutter**, Android/iOS). |
| `import_service_server` | Backend (**Node.js**, Fastify): REST API, БД, JWT, интеграция с 1С, WebSocket для чата. |
| `import_service_admin` | Панель администратора (**Flutter Web**). |

## Технологии

Flutter/Dart; сервер — Node.js 18+, MySQL. Переменные окружения и секреты задаются локально (см. `import_service_server/.env.example`), в репозиторий секреты не коммитятся.

## Публикация изменений на GitHub

После правок в **любом** из трёх каталогов коммит и push выполняются из **корня** монорепозитория:

```powershell
cd <путь_к_клону>\import_servis
.\scripts\push-monorepo.ps1 -Message "Краткое описание изменений на русском"
```

Дополнительно см. `.cursor/rules/git-push-monorepo.mdc` внутри каждого проекта.
